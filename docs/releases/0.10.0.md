# Obsidian Amazing Marvin 0.10.0

0.10.0 makes Amazing Marvin, Obsidian, and an agent usable as one workflow:

- Use **Amazing Marvin** to plan and hold the task state.
- Use **Obsidian Amazing Marvin** to keep the useful Marvin structure and task
  views in a vault without overwriting the notes you add around them.
- Use the **Amazing Marvin MCP** when an agent needs to read Marvin, discover a
  project, create work, or complete it without going through the plugin or
  mutating the vault.

The outcome is a shared workflow with clear boundaries: people keep context in
notes, agents use Marvin's stable IDs, and both work against the same Marvin
task state. The plugin owns vault projections; the MCP owns Marvin-only
operations.

## What this enables

- Keep context and prose beside an imported project without losing it on the
  next import.
- Ask an agent to find the right Marvin project, create a task in it, and read
  the result back without making the agent scrape an Obsidian note or guess a
  parent ID.
- Use a daily note as an execution surface that stays current without rerunning
  the whole template.
- Limit what reaches the vault and choose a task format that works with the
  rest of your Obsidian setup.

## What changed

### Imports preserve your notes

- Category, project, and Inbox imports now refresh only the plugin-managed
  region of a note. Your prose and custom frontmatter outside that region stay
  intact.
- Older imported notes are adopted when their generated Marvin content can be
  identified. Future imports use explicit markers.
- The importer repairs the older malformed YAML list syntax before writing
  native YAML lists.
- Change the managed folder in settings. Imported notes move by their Marvin
  ID when possible; collisions stop the import instead of overwriting a file.
- Notes for Marvin items you no longer import remain in the vault for you to
  review or remove yourself.

### Choose what to import

- Import all categories/projects, or select one or more roots.
- A selected root includes its descendants. Ancestors remain as navigation-only
  notes so links and hierarchy still make sense.
- Inbox import is independent of category/project selection.
- Deselecting a root never deletes notes from an earlier import.

### Better day-to-day task work

- Refresh a bounded Today region in a daily note instead of rerunning a whole
  template. The region can refresh on startup, focus, and an interval after
  you initialize it once.
- Create a task from an imported category/project note and the plugin defaults
  it to that Marvin parent. This also works for the selected-text shortcut.
- Choose the current Dataview metadata, Tasks Dataview fields, or Tasks emoji
  dates. You can put titles first, format date links for weekly notes, add a
  Tasks query tag, and project Marvin labels as namespaced Obsidian tags.
- Optional completion sync still marks a linked Marvin task done when you
  check it off in Obsidian.

### Companion MCP server for agents

The repository now includes a local stdio MCP server for Marvin-only agent
work. An agent can read today/due work, discover categories, children, and
labels, create tasks, and mark work complete. It uses stable Marvin IDs instead
of inferring a project from a title.

The MCP and plugin share the same typed client. Reads can try the Amazing
Marvin desktop API first and fall back to the public API; cached and stale
responses report their freshness and origin. Route failures and malformed
dates return a structured error envelope for callers to inspect.

The MCP does not edit the vault. When a workflow must both create a task and
record its source note, use the plugin's in-Obsidian automation API instead.

See [Companion MCP server](../../README.md#companion-mcp-server) for build,
secret-handling, and registration instructions. Use a local launcher or secret
manager to provide the limited Marvin API token; do not put it directly in an
MCP-host configuration file.

## Upgrade notes

- This release keeps Amazing Marvin as the source of truth for imports. It
  does not sync arbitrary edits from Obsidian back to Marvin.
- Existing imported notes should be preserved, but the first import after
  upgrading may add a managed marker around recognizable generated content.
- The MCP does not edit the vault. Use the in-Obsidian automation API when a
  workflow must create a Marvin task and record its source-note association
  together.
- The minimum supported Obsidian version is now 1.12.7.

## Verification

The release was exercised against a real Marvin account and Obsidian vault:
non-destructive import preservation, context-aware task creation, task
completion, local-first MCP reads, MCP category/child discovery, task
creation/completion, cache freshness, and structured malformed-date errors.
